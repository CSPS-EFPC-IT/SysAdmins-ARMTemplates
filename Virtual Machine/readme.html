Virtual Machine

This section includes two parts: an ARM template which deploys a Windows server virtual machine and the other which deploys a Linux virtual machine.

Windows Server Virtual Machine

The VM_Windows.json ARM template can be used to deploy a virtual machine with an OSdisk, nic and public IP. Feel free to use the VM_Windows.parameters.json as parameter examples.

Note: If you wish to create a Linux hosted virtual machine, you may proceed to the Linux virtual machine section below.

  1. As mentioned above, use the VM_Windows.json file (ARM template) in the current folder.
  2. Pass in the following parameters:
    • projectName - The name of the project/solution. This string will be used to name the virtual network.
  3. The file will output the username of the admin account.

Ubuntu 18.04 LTS Virtual Machine

The VMUbuntu1804LTSDeploys.json ARM template can be used to deploy a virtual machine with an OSdisk, nic and public IP. Feel free to use the VMUbuntu1804LTSDeploys.parameters.json as parameter examples.

  1. As mentioned above, use the VMUbuntu1804LTSDeploys.json file (ARM template) in the current folder.
  2. Pass in the following parameters:
    • subnetName: The name of the pre-created subnet. ex: example-SN
    • vnetName:The ID of the virtual network.
    • publicIpAddressType: The type of the public ip. ex: static or dynamic
    • publicIpAddressSku: The SKU of the public ip. ex: basic or standard
    • virtualMachineSize: The size of the VM. ex: Standard_B2s
    • adminUsername: The username of the default admin account
    • adminPassword: The password of the default admin account
    • diagnosticsStorageAccountName: The name the storage account for diagnostics. (24 character max, format: [ShortProjectName][env]"diagsa"
    • projectName: The name of the project. This string will be used to name the virtual machine and associated resources.
    • vmPurpose: description: The purpose of the virtual machine. ex: App (application), Data (database)
    • vmNumber: The number of the virtual machine. This integer will be used to name the virtual machine. (ex: 01)
    • environmentName: The environment of the virtual network. This string will be used to name the network security group.
  3. The file will output the username of the admin account.
